01 . what is diff between arm64 and amd64

    arm64 and amd64 are different CPU architectures:
    amd64:
    Also called x86_64
    Used by Intel and AMD desktop/server CPUs
    Very common in laptops, desktops, cloud VMs

    arm64:
    Also called aarch64
    Used by ARM CPUs (Apple Silicon, AWS Graviton, many mobile/embedded devices)
    Often better power efficiency (performance per watt)
    Key difference: binaries are architecture-specific.
    An amd64 binary won’t run natively on arm64 (and vice versa) unless you use emulation or build multi-arch images/binaries.

02. what is containerd
    containerd is a container runtime daemon.

    In practical terms, it is the software that actually manages container lifecycle on a node:
    pull images
    unpack images
    create/start/stop containers
    manage container storage and snapshots
    In Kubernetes, kubelet talks to containerd through the CRI (Container Runtime Interface).
    So Kubernetes schedules pods, and containerd does the low-level work to run their containers.

10. CA [Certificate Authority] 
    A Certificate Authority (CA) is a trusted entity that issues and signs digital certificates.

    In Kubernetes:
    The CA signs certs for components (API server, kubelet, etcd, users).
    Components trust the CA certificate.
    If a cert is signed by that CA, it’s treated as authentic.
    So CA provides:

    Identity verification (who is this component/user?)
    Trust chain (signed by trusted CA)
    Basis for TLS encryption + mutual authentication (mTLS)

13. kubelet? 
    kubelet is the Kubernetes node agent that runs on every worker (and often control-plane) node.
    It:
    * watches Pod specs assigned to its node (from API server)
    * asks container runtime (containerd) to start/stop containers
    * mounts volumes/secrets/configs
    * runs health checks and reports node/pod status back to API server
    * So scheduler decides where a pod runs, and kubelet makes it actually run on that node.
